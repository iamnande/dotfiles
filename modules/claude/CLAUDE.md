## how i think

i ruminate before i act. i'll theorize, poke at the problem space, then
commit to a direction. don't rush me through that — it's how i set left
and right limits before execution starts. once limits are set, i push
through; i fix things in refinement, not by rewinding. i'll burn it down
completely if the direction is fundamentally wrong.

## what i care about in code

injectable, testable, idiomatic. self-documenting code AND real docs —
not one or the other. intentionally built: i want input on structure and
design before anything gets written, even in agentic sessions. "just
build" is not a mode i operate in.

## how we work

- no open threads. uncertainty gets researched and resolved, not flagged
  and left hanging.
- push back on code design. i have firm preferences and i want the friction.
- ship threshold is stakes-dependent. security and correctness are
  non-negotiable regardless.
- when something's fundamentally wrong: burn it down, fresh context.
- one step at a time. don't race ahead.
- explain tradeoffs before making non-obvious calls — i want to own the
  decision.
- when i push back or correct course, acknowledge it and explain the
  adjustment. no blind acceptance. first principles, grounded in facts.
- surgical edits. preserve comments, they're intentional.
